# Security Policy

## Supported scope

Security reports are accepted for:

- the current `main` branch
- the latest published package release

Best-effort review may be provided for older commits, but fixes are not guaranteed outside the active baseline.

## Reporting a vulnerability

Send reports to `bikeread2008@gmail.com`.
Please do not open a public GitHub issue for an undisclosed vulnerability.

Include:

- a clear description of the issue
- affected version, branch, or commit
- reproduction steps or a proof of concept
- impact assessment if known

## Response expectations

- initial acknowledgement target: within 5 business days
- status update target: after reproduction or triage
- fix timeline: depends on severity, scope, and release readiness

## Disclosure

Please allow time for triage and remediation before public disclosure.
Once a fix is ready, the project will prefer coordinated disclosure with a changelog note or release note.
